配信システム比較検討

チケット申込 × オンデマンド配信システムの3案を比較

現行 案1

EventRegist + Enavle

統合型（同一ベンダー）

初期費用55万円〜
決済手数料8%
請求書払い対応
代理視聴登録不可
Enavle海外サーバーのため、各参加者が本人登録必須
低コスト 案2

Peatix + OneStream

分離型（別サービス）

初期費用ほぼ無料
決済手数料約5%
請求書払い非対応
代理視聴登録手動対応
事務局の手動作業が多い（情報収集・アカウント登録）
推奨 案3

CMS + 内製

WordPress/microCMS + Cloudflare Stream

初期費用開発費依存
決済手数料3.6%（Stripe）
請求書払い実装可能
代理視聴登録自動化可能
申込→視聴までシームレス。長期的にコスト最小

事務局コメント（運用実績）

過去数年間、案1（EventRegist + Enavle）を採用して運用してきた結果：

  • Enavleプレーヤーの評判が悪い - 参加者から不満・クレームが発生
  • 改善要望が反映されない - 複数回フィードバックしても目立った改善なし
  • 高額なコストに見合う価値か疑問 - 55万円〜/イベント

→ システムの見直しを検討する時期に来ている

結論

短期的（次回開催）

案2（Peatix + OneStream）で低コスト・低リスクにスタート。 手動作業は発生するが、現行より参加者体験は改善。

中長期的（2〜3年後）

案3（内製）へ段階移行。 決済手数料最小化、代理申込の自動化、ブランド体験の統一を実現。

いずれの場合も、案1（現行）の継続は非推奨。
高コスト・低品質・改善されないという三重苦からの脱却が必要。
